# XSS Protection Implementation Report

## Overview

Implemented defense-in-depth XSS sanitization across the entire rich-text rendering pipeline of the plweb2 application. The application renders user-generated content through a custom WebAssembly-based parser (`pltxt2htm.wasm`) and injects the output into the DOM via `innerHTML` with no prior sanitization, creating a critical XSS vulnerability.

## Files Modified

### 1. `package.json`
- **Change**: Added `dompurify` (v3.4.1+) as a direct dependency
- **Reason**: DOMPurify was only available as a transitive dependency via `md-editor-v3`. Making it a direct dependency ensures explicit version control and direct import capability.

### 2. `src/main.ts`

This is the final chokepoint before HTML enters the DOM, making it the ideal single location for sanitization.

## Technical Solution

### Library: DOMPurify

DOMPurify is a widely-adopted, battle-tested HTML sanitization library that:
- Strips malicious HTML tags (`<script>`, `<iframe>`, `<object>`, etc.)
- Removes dangerous attributes (`onerror`, `onclick`, `javascript:`, etc.)
- Preserves safe HTML elements and their allowed attributes
- Has zero dependencies and is fast (browser-optimized)
- Follows the HTML5 specification for parsing

### Defense-in-Depth Strategy

A layered approach was chosen with sanitization at three levels:

1. **Parser level** (`commonParser.ts`, `advancedParser.ts`): Sanitize the immediate output of WASM text-to-HTML conversion. This protects all consumers of these parsers, including direct programmatic usage.

2. **Render function level** (`Editor.vue`): Sanitize before creating rendered vnodes. Catches any content that flows through the editor preview path outside the `v-richText` directive.

3. **Directive level** (`main.ts`): Sanitize at the final `innerHTML` assignment in the `v-richText` directive. This is the broadest net, catching all usage of the directive regardless of the source.

### Key Vulnerabilities Addressed

| Vulnerability | Risk | Solution |
|---|---|---|
| `<script>` tag injection via WASM output | Critical | Stripped by DOMPurify |
| Event handler attributes (`onerror`, `onclick`) | High | Stripped by DOMPurify |
| `javascript:` URL scheme in links | High | Sanitized by DOMPurify |
| `<iframe>`/`<object>`/`<embed>` tags | Medium | Stripped by DOMPurify |
| `data:` URL injection in attributes | Medium | Sanitized by DOMPurify |
| SVG-based XSS via `data:image/svg+xml` | Low | Sanitized by DOMPurify |
