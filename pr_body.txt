### Detection Criteria
For each payload, the WASM parser output was analyzed for:
1. **Unescaped `<script>` tags** — Any occurrence of raw `<script>` in output
2. **Unescaped event handler attributes** — Any raw `onclick`, `onerror`, `onload`, etc. on HTML elements
3. **`javascript:` URLs** — Any `href` or `src` attribute containing `javascript:` scheme
4. **Unescaped dangerous HTML tags** — `<svg>`, `<math>`, `<body>`, `<input>`, `<details>`, `<object>`, `<embed>`
5. **Attribute breakout** — Whether quote characters in markdown link/image URLs escape the attribute context

## Technical Analysis

### How the WASM Parser Prevents XSS

The pltxt2htm C++ WASM module employs multiple defense layers:

1. **HTML entity encoding**: All HTML special characters (`<`, `>`, `&`, `"`, `'`) in text content are encoded to their entity equivalents (`&lt;`, `&gt;`, `&amp;`, `&quot;`, `&apos;`). Raw HTML tags like `<script>`, `<img>`, `<svg>` become `&lt;script&gt;`, `&lt;img&gt;`, `&lt;svg&gt;` and are rendered as safe text.

2. **Strict tag whitelist**: Only ~30 structural HTML tags are emitted by the parser (e.g., `<p>`, `<em>`, `<strong>`, `<code>`, `<pre>`, `<table>`, `<a>`, `<img>`). These tags are produced from the parser's own markdown processing, never from raw user HTML input.

3. **URL scheme validation**: Only `http://` and `https://` URL schemes are accepted. `javascript:`, `data:`, `vbscript:`, and other dangerous schemes are rejected.

4. **Percent-encoding of dangerous characters in URLs**: When markdown link syntax is used (`[text](url)`), dangerous characters like `"`, `'`, `>`, `<` in the URL are percent-encoded, preventing attribute breakout attacks.

5. **No generic attribute parsing**: The parser does not support arbitrary HTML attributes. Event handlers like `onclick`, `onerror`, `onload` cannot be injected because the parser doesn't have a mechanism to emit attribute name-value pairs from user input (except for `href` on `<a>` and `src` on `<img>` which are URL-validated).

6. **Post-processing safety**: After WASM parsing, the `advancedParser.ts` applies KaTeX (with safe `ignoredTags` configuration), Mermaid (with `securityLevel: 'strict'`), and highlight.js. None of these introduce XSS vulnerabilities in their current configurations.

### Specific Attack Vector Results

| Attack Vector | Payload Example | WASM Output | Verdict |
|---|---|---|---|
| Script tag injection | `<script>alert(1)</script>` | `&lt;script&gt;alert(1)&lt;/script&gt;` | Safe |
| Event handler | `<img src=x onerror=alert(1)>` | `&lt;img&nbsp;src=x&nbsp;onerror=alert(1)&gt;` | Safe |
| SVG onload | `<svg onload=alert(1)>` | `&lt;svg&nbsp;onload=alert(1)&gt;` | Safe |
| javascript: URL in link | `[click](javascript:alert(1))` | `[click](javascript:alert(1))` (markdown not rendered) | Safe |
| Attribute breakout | `![x](http://a.com" onerror="alert(1))` | `src="http://a.com%22%20onerror=%22alert(1"` | Safe (percent-encoded) |
| Polyglot (multi-vector) | `jaVasCript:/*-/*`... | All HTML entities properly encoded | Safe |
| Null byte injection | `<scr<script>ipt>alert(1)` | `&lt;scr&lt;script&gt;ipt&gt;alert(1)` | Safe |
| Unicode/BOM prefix | `\uFF00<script>alert(1)</script>` | `\uFF00&lt;script&gt;alert(1)&lt;/script&gt;` | Safe |
