## Technical Solution

The implementation follows Vue 3 Composition API patterns with `<script setup>` and TypeScript as used throughout the codebase:

1. **Query-Param-Driven Navigation**: The feature uses URL query parameters (`from`, `skip`, `category`) to encode the exact notification position. This allows deep linking to specific notifications and maintains scroll position across navigation.

2. **Router Integration**: Uses `vue-router`'s `useRoute` and `useRouter` composables for reading/writing query params and for SPA navigation. Navigation uses named routes (`notifications`, `Comments`, `ExperimentSummary`, `profile`) for type safety.

3. **DOM Scrolling via `scrollIntoView`**: After loading notification items, the target message is located in the DOM using the `data-message-id` attribute and scrolled into view using the native `scrollIntoView` API with smooth animation.

4. **Tab Synchronization**: The `Notifications.vue` page reads the `category` query param to determine which tab should be active. Each `NotificationList` instance checks if it belongs to the active tab before updating route query params.

5. **Progressive Pagination Tracking**: As the user scrolls through notifications (infinite scroll), the `skip` query parameter is updated to reflect the current offset, enabling URL-based position restoration on page reload.

## Summary

The implementation enables clicking any notification item to navigate to its context page (comments page for replies, profile for followers, experiment page for works, or the notifications page with `from`/`category` params for system/admin messages). The notifications page reads `from`, `skip`, and `category` URL query parameters to automatically switch to the correct tab, load the appropriate page offset, and smoothly scroll to the exact notification message. Route params are updated in real-time as pagination progresses, making notification positions shareable and restorable via URL.
